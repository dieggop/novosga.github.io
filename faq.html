FAQ

Seção com algumas das perguntas feitas frequentemente.

Caso não encontre sua dúvida aqui, sinta-se à vontade para publicá-la no nosso fórum.

O Novo SGA funciona no Windows?

Sim. O Novo SGA é escrito em PHP e Java, que são linguagens multiplataformas.

Porém, a documentação do Novo SGA é focada em distribuíções Linux baseadas em Debian.

Qual distribuição Linux devo utilizar?

Qualquer distribuíção que tenha suporte à instalação do PHP 5.3+ e Java 7.

Porém, a documentação do Novo SGA é focada em distribuíções Linux baseadas em Debian.

Quais os requisitos de hardware para instalar o Novo SGA?

Definir os requisitos mínimos de hardware para sustentar o sistema depende de vários fatores, como por exemplo o tamanho da instituição, a quantidade de unidades de atendimento e de usuários, número de acessos simultâneos, volume de atendimento e etc.

Existe alguma forma de imprimir direto sem escolher a impressora?

Isso varia de acordo com o navegador que esteja utilizando.

Firefox

No firefox há uma opção chamada "silent print".

  1. Acesse a página about:config
  2. clique com o botão direito em qualquer lugar da página e selecione "Novo->Booleano".
  3. digite o nome "print.always_print_silent"
  4. selecione o valor "verdadeiro" (true)

Chrome 18+

No chrome pode imprimir silenciosamente utilizando a opção kiosk. Basta adicionar os parâmetros "--kiosk --kiosk-printing" no atalho do navegador

  • Linux: google-chrome --kiosk --kiosk-printing
  • Windows: botão direito no atalho, propriedades, e adicionar os parâmetros (--kiosk --kiosk-printing) ao final do campo alvo.

Qual é o modelo de impressora recomendado?

Impressora térmicas, pois são mais econômicas.

Estou tendo problemas com a codificação dos caracteres. O que pode ser?

O encoding utilizado pelo Novo SGA é o UTF-8.

O que pode estar acontecendo é que o banco de dados (banco ou tabela) não em UTF-8.

MySQL

Se você estiver utilizando o MySQL, verifique se o arquivo my.conf está configurado como mostra abaixo:

[client]
...
default-character-set = utf8

[mysqld]
...
character-set-server = utf8

Erro "Falha obtendo unidades". O que pode ser?

A principal razão para não conseguir obter unidades é causada por erros de configuração (server.conf) do controlador de paineis.

Sugiro que revisem com cuidado esse arquivo, atentando para todos os detalhes. Os comentários contidos nesse arquivo devem ser lidos e entendidos para que não hajam equívocos.

Também devem ser considerados outros problemas alheios ao sistema, como problemas de rede, firewall etc.

Problemas com bloqueio por proxy também já foram relatados.

Como alterar a palavra "Guichê" que aparece no painel?

Para versões até 0.7.2

Abra o arquivo core/business/AtendimentoBusiness.php, procure por nm_local, e altere o valor desse parâmetro para o nome desejado.

No caso abaixo, ao invés de "Guichê" aparecerá "Sala":

// Antes
$stmt->bindValue('nm_local', _('Guichê'));
// Depois
$stmt->bindValue('nm_local', _('Sala'));

Apareceu mensagem de nova versão disponível. Como atualizo o Novo SGA?

Primeiramente é bom acessar o blog do Novo SGA para saber quais mudanças foram feitas. Como o sistema é dividido em 3 partes, nem sempre as 3 sofrem anterações. Depois basta ir à página de download e baixar a nova versão

Atualizando o Painel

Para atualizar o painel (tanto cliente quanto servidor) basta apenas parar o processo e substituir os arquivos pela nova versão.

Atualizando a parte Web

Já na parte web, é necessário manter o arquivo de configuração atual. Para isso basta seguir os passos:

  • Extraia a versão web em um novo diretório
  • Substitua o arquivo core/Config.php pelo o mesmo arquivo da versão já instalada
  • Teste a nova versão e depois substitua a antiga pela nova. Pronto.
